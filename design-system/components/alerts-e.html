Purpose of alerts

Alerts are used to call attention to information that needs to stand out from the rest of the page content. Use to provide contextual feedback messages for typical user actions. Choose from a handful of available and flexible alert messages.

Design options

Basic alerts

Use to wrap any text and an optional dismiss button in .alert, or one of the four contextual classes (for example, .alert-success) for basic alert messages.

Alerts don't have default classes, only base and modifier classes. A default gray alert doesn't make too much sense, so always specify a type with a contextual class. Choose from success, info, warning, or danger.

Correct use

General
  • Place in the logical flow of a page
  • Ensure it relates to the content immediately around it
  • Begin <section> with a heading that is specific to the featured content, and contained within it
  • Use a general heading only if a specific heading is not possible (for example, "Congratulations", "Summary", "Attention", etc.)
  • Do not use <section> within another section, or validation issues occur
    • Use <div> to style the alert
Success alert
  • Use .alert-success to acknowledge something positive for example, a completed task, something that is correct
  • With the Success alert, the title appears next to a checkmark icon; the content background is green.
Info alert
  • Use .alert-info to highlight fairly important information
  • With the Info alert, the title appears next to a letter "i" icon; the content background is blue
  • Use to highlight targeted information for specific segment groups
Warning alert
  • Use .alert-warning to warn the user
  • With the Warning alert, the title appears next to a caution icon; the content background is yellow.
Danger alert
  • Use .alert-danger for very important, negative or dangerous, information/actions for example, systems that are down, pending changes to X, "Are you sure you want to delete X?"
  • With the Danger alert, the title appears next to an exclamation icon; the content background is red.
Alert link
  • To override the default appearance of a link, use the .alert-link utility class to quickly provide matching coloured links within any alert
