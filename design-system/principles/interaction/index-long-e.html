Principle of visibility

Definition: Users need to easily find and see ways to use the page. The options are visible and are not distracting them with unnecessary or redundant information.

In practice: A good design will not confuse or overwhelm the user with unnecessary alternatives.

Designers will naturally understand this as the basic hierarchy of a design. Visibility isn’t a new idea... The key is to prioritize... If we design by committee and attempt to make everything important, we will in effect emphasize nothing.

Taking Donald Norman’s Design Principles to Web Design (external link)
Patrick McNeil

Principle of feedback

Definition: User has to be informed of the outcome of their action immediately after it occurs.

In practice: The design needs to provide feedback mechanisms for events/actions for example, a change of state or errors. Feedback must be clear, concise, in plain language.

Principle of affordances

Definition: The visual design properties of an object should suggest how to use it.

In practice: Links need to look and behave like links, including their default, hover, active and visited states. Buttons needs to look and behave like buttons, including their default, hover, and active states. Images and icons that are clickable need to look like they are clickable. All page elements that have an action to them, must suggest/imply the action. The actions must match the user's expectations.

Principle of mapping

Definition: There should be a relationship between controls and their effect. The user can then simply recognize how to use the system rather than have to recall, like on a stove. This reduces reliance on memory which slows down ability to act. Recognition is faster than recall. For example, the vertical scroll bar on a page tells the user where they are on the page. As it is dragged down, the page moves down at the same rate.

In practice: Position controls so that their locations and behaviours correspond to the layout and behaviours. For example, paging forward is on the right, and paging back is on the left. Additionally, if direction is implied thought arrows, it matches the user's mental model of back/forward/up/down.

Principle of constraints

Definition: There should be restrictions built into the interaction to reduce user error.

In practice: Where possible, prevent user errors by:

  • Disabling functions that do not apply
  • Using appropriate controls to constrain inputs (for example, radio buttons, drop-downs)

Principle of consistency

Definition: Similar actions and similar elements should be used to accomplish similar tasks (this reduces error and improves learnability). Consistency applies across all system levels and includes both aesthetic and functional elements:

  • Application (consistency from screen to screen or page to page)
  • Device (consistency on desktop, tablet, mobile device)
  • Platform (consistency across the whole system such as internal or external site)
  • Industry (consistency within the government domain)

In practice: In addition to matching the user's expectations in terminology, layout, and interactions, the experience should be consistent throughout the entire process and between related pages.

Put your buttons in consistent places on all your windows, use the same wording in labels and messages, use consistent color scheme throughout, and any other consistent models that are applicable for you.

Reuse Principle in User Interfaces (external link) Vishal Mehta
